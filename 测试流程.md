# 操作系统模拟器 测试流程

> 运行前先确保 `生成/` 下没有旧的 `数据/os_state.bin`，或直接删掉 `生成/数据/` 目录从零开始。

## 一、用户管理

### 1.1 注册新用户

```
register admin 123456
```

预期输出：
```
注册成功! 用户名: admin
```

### 1.2 重复注册

```
register admin 123456
```

预期输出：
```
注册失败: 用户名 admin 已存在
```

### 1.3 登录

```
login admin 123456
```

预期输出：
```
登录成功! 欢迎 admin
```

### 1.4 错误密码

```
login admin 111111
```

预期输出：
```
登录失败: 密码错误！剩余尝试次数: 2
```
> 连续3次输错会锁定账户。

### 1.5 未登录执行命令

```
logout
create_pcb test 5
```

预期输出：
```
用户 admin 已登出
请先登录! 使用 login <用户名> <密码>
```

## 三、调度器

```
create_pcb high 0 1 5
create_pcb mid 4 1 5
create_pcb low 8 1 5
```
> 显式指定 ppid=1、burstTime=5，确保行为可预测。

### 3.1 启动调度器

```
start_sched
```

调度器每 500ms 自动调度一次，会在提示符之间输出调度日志。等待约 3~5 秒后自动停止。预期会看到类似输出：

```
调度器已启动 (多级反馈队列)
Q0(优先级 0-3): high(7)
Q1(优先级 4-7): mid(8)
Q2(优先级 8-15): low(9)

[调度] 选中 PID=7 (high), 队列Q0, 时间片=1, CPU=1/5
[调度] 选中 PID=8 (mid), 队列Q1, 时间片=2, CPU=2/5
[调度] 选中 PID=7 (high), 队列Q1, 时间片=2, CPU=3/5
...
[调度] 所有用户进程已完成, 调度器自动停止
```
> 注意：具体 PID 取决于当前 nextPid，上面假设前面已清空，pid 从 7 开始。实际序号可能不同。

### 3.2 暂停调度

```
stop_sched
```

预期输出：
```
调度器已停止
```

### 3.3 单步执行

先创建测试进程：
```
create_pcb step_a 3 1 3
create_pcb step_b 7 1 3
```

将进程加入队列（先 start_sched 再立即 stop，或直接 step——step 会自动补充队列）：
```
step
step
step
step
```

每次 step 输出类似：
```
[调度] 选中 PID=10 (step_a), 队列Q0, 时间片=1, CPU=1/3
[调度] 选中 PID=11 (step_b), 队列Q1, 时间片=2, CPU=2/3
[调度] 选中 PID=10 (step_a), 队列Q1, 时间片=2, CPU=3/3
[完成] PID=10 (step_a) 已执行完毕, 自动终止
```
> step 过程中进程可能降级到下一队列。跑完 burstTime 自动终止。

### 3.4 重启调度

```
restart_sched
```

预期输出：
```
调度器已重启
```
> 重启后队列清空，调度器从当前就绪进程重新入队。

```

```

## 七、IPC 多实例（可选，需 Windows）

```
# 终端1（后端 — 先启动）
```

## 八、异常/边界测试

### 8.1 非法参数

```
create_pcb              → 用法: create_pcb <进程名> <优先级(0-15)> [父进程PID] [需要CPU时间]
create_pcb x 20         → (优先级20会被截断为15, 正常创建)
renice 999 5            → 修改失败: PID不存在或优先级范围无效(0-15)
kill_pcb 999            → 撤销失败: PID=999 不存在或为init进程
alloc -1 1              → 大小必须在1-1024KB之间
```

### 8.2 边界值

```
create_pcb edge_a 0         → 优先级0（最高）
create_pcb edge_b 15        → 优先级15（最低）
```

预期输出：
```
进程创建成功! PID=17 名称=edge_a 优先级=0 需要CPU=5
进程创建成功! PID=18 名称=edge_b 优先级=15 需要CPU=5
```

清理：
```
kill_pcb 17
kill_pcb 18
```

### 8.3 kill init

```
kill_pcb 1
```

预期输出：
```
撤销失败: PID=1 不存在或为init进程
```
