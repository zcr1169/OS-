## 三、调度器（核心演示）

> 本节是验收重点。构建 CPU 密集型进程，覆盖三个队列，直观展示升降级和交替运行。

先确保前面进程全部清干净：

```bash
kill_pcb 3
kill_pcb 4
```

### 3.1 构建三个 CPU 密集型测试进程

```bash
create_pcb worker_A 2 1 10
create_pcb worker_B 5 1 10
create_pcb worker_C 10 1 10
```

三个进程 burstTime=10（需要 10 个 CPU 时间才能跑完），优先级分布在三个队列：

- worker_A 优先级 2 → Q0（时间片 1）
- worker_B 优先级 5 → Q1（时间片 2）
- worker_C 优先级 10 → Q2（时间片 4）

### 3.2 自动调度 —— 观察完整降级链

```bash
start_sched
```

每 500ms 自动输出一行调度日志。约 10~15 秒跑完，关键输出：

```text
调度器已启动 (多级反馈队列)
Q0(优先级 0-3): worker_A(2)
Q1(优先级 4-7): worker_B(3)
Q2(优先级 8-15): worker_C(4)

[调度] 选中 PID=2 (worker_A), 队列Q0, 时间片=1, CPU=1/10
```

> 决策：Q0 非空 → 取 worker_A → 时间片=1 → 1<10 没跑完 → 降级到 Q1

```text
[调度] 选中 PID=2 (worker_A), 队列Q1, 时间片=2, CPU=3/10
```

> Q0 空 → Q1 取 worker_A → 时间片=2 → 3<10 → 降级到 Q2

```text
[调度] 选中 PID=3 (worker_B), 队列Q1, 时间片=2, CPU=2/10
```

> Q0 空 → Q1 取 worker_B → 时间片=2 → 2<10 → 降级到 Q2

```text
[调度] 选中 PID=2 (worker_A), 队列Q2, 时间片=4, CPU=7/10
```

> Q0、Q1 均空 → Q2 队首 worker_A → 时间片=4 → 7<10 → 已在 Q2，不降

```text
[调度] 选中 PID=3 (worker_B), 队列Q2, 时间片=4, CPU=6/10
[调度] 选中 PID=4 (worker_C), 队列Q2, 时间片=4, CPU=4/10
[调度] 选中 PID=2 (worker_A), 队列Q2, 时间片=4, CPU=11/10
[完成] PID=2 (worker_A) 已执行完毕, 自动终止
[调度] 选中 PID=3 (worker_B), 队列Q2, 时间片=4, CPU=10/10
[完成] PID=3 (worker_B) 已执行完毕, 自动终止
[调度] 选中 PID=4 (worker_C), 队列Q2, 时间片=4, CPU=8/10
[调度] 选中 PID=4 (worker_C), 队列Q2, 时间片=4, CPU=12/10
[完成] PID=4 (worker_C) 已执行完毕, 自动终止

[调度] 所有用户进程已完成, 调度器自动停止
```

**验收要点**：

- worker_A 经历了 **Q0 → Q1 → Q2** 完整三次降级
- worker_B 从 Q1 → Q2，worker_C 一直在 Q2
- 三个进程交替运行，时间片大的等得久但一次跑得多
- CPU 数值在动态增长，不是直接改状态

### 3.3 单步演示 —— 完整决策链路

> 核心要求：每一步必须展示「当前队列 → 选中谁 → 时间片 → 跑完没 → 去哪」的完整推导。

清空旧进程，创建新测试进程。burstTime=6 确保多步才能跑完：

```bash
kill_pcb 2
kill_pcb 3
kill_pcb 4
create_pcb s_a 3 1 6
create_pcb s_b 8 1 6
```

s_a 优先级 3→Q0（时间片 1），s_b 优先级 8→Q2（时间片 4）。

#### 第一步

```bash
step
```

```text
[调度] 选中 PID=5 (s_a), 队列Q0, 时间片=1, CPU=1/6
```

> **决策推导**：扫描 Q0=[s_a], Q1=[], Q2=[s_b]。Q0 非空 → 取 s_a → 时间片 1 → CPU 1 < 6 未完成 → 降级到 Q1。现在 Q1=[s_a], Q2=[s_b]。

#### 第二步

```bash
step
```

```text
[调度] 选中 PID=5 (s_a), 队列Q1, 时间片=2, CPU=3/6
```

> **决策推导**：Q0=[], Q1=[s_a], Q2=[s_b]。Q1 队首 s_a → 时间片 2 → CPU 3 < 6 → 降级到 Q2。现在 Q2=[s_a, s_b]。

#### 第三步

```bash
step
```

```text
[调度] 选中 PID=5 (s_a), 队列Q2, 时间片=4, CPU=7/6
[完成] PID=5 (s_a) 已执行完毕, 自动终止
```

> **决策推导**：Q0=[], Q1=[], Q2=[s_a, s_b]。Q2 队首 s_a → 时间片 4 → CPU 7 ≥ 6 → 完成！s_a 自动终止。队列剩下 Q2=[s_b]。

#### 第四步

```bash
step
```

```text
[调度] 选中 PID=6 (s_b), 队列Q2, 时间片=4, CPU=4/6
```

> Q2 只剩 s_b，4 < 6 继续跑，已在 Q2 不降级。

#### 第五步

```bash
step
```

```text
[调度] 选中 PID=6 (s_b), 队列Q2, 时间片=4, CPU=8/6
[完成] PID=6 (s_b) 已执行完毕, 自动终止
```

#### 第六步

```bash
step
```

```text
所有调度队列为空，无可调度进程
```

### 3.4 restart vs start 的区别

```bash
create_pcb r_a 0 1 4
create_pcb r_b 4 1 4
start_sched
# 自动跑完后 stop_sched
stop_sched
create_pcb r_c 8 1 4
restart_sched
```

`restart_sched` 保留队列状态和调度计数，在已有队列基础上继续。`start_sched` 会扫描所有就绪进程重建队列。

---

### 3.5 调度中 step（无需先 stop）

调度器运行时直接敲 `step`：

```bash
start_sched
step
```

自动调度和手动 step 共享同一把锁，可以穿插使用，验收时不用频繁 stop/start。

### 验证3：B 端发命令，A 端执行，两端同步生效


**窗口1（后端）** — 确认后端也有这个进程：

## 八、异常/边界测试

### 8.1 非法参数

```bash
create_pcb              → 用法: create_pcb <进程名> <优先级(0-15)> [父进程PID] [需要CPU时间]
create_pcb x 20         → (优先级20会被截断为15, 正常创建)
renice 999 5            → 修改失败: PID不存在或优先级范围无效(0-15)
kill_pcb 999            → 撤销失败: PID=999 不存在或为init进程
alloc -1 1              → 大小必须在1-1024KB之间
```

### 8.2 边界值

```bash
create_pcb edge_a 0         → 优先级0（最高）
create_pcb edge_b 15        → 优先级15（最低）
```

预期输出：

```text
进程创建成功! PID=17 名称=edge_a 优先级=0 需要CPU=5
进程创建成功! PID=18 名称=edge_b 优先级=15 需要CPU=5
```

清理：

```bash
kill_pcb 17
kill_pcb 18
```

### 8.3 kill init

```bash
kill_pcb 1
```

预期输出：

```text
撤销失败: PID=1 不存在或为init进程
```
