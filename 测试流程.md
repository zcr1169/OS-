> 运行前先确保清空旧的持久化数据：`del /q 生成\数据\os_state.bin 2>nul`

> 数据文件夹（`生成/数据/`）始终位于 exe 所在目录，从**项目根目录**或 `cd 生成\` 启动均可。

## 三、调度器（核心演示）

> 本节是验收重点。构建 CPU 密集型进程，覆盖三个队列，直观展示升降级和交替运行。
> 
> 从第二节结束时的状态继续（进程已全部清空，仅剩 init）。若当前有残留进程，先用 `kill_pcb` 清理。

### 3.1 构建三个 CPU 密集型测试进程

```bash
create_pcb worker_A 2 1 10
create_pcb worker_B 5 1 10
create_pcb worker_C 10 1 10
```

三个进程 burstTime=10（需要 10 个 CPU 时间才能跑完），优先级分布在三个队列：

- worker_A 优先级 2 → Q0（时间片 2）
- worker_B 优先级 5 → Q1（时间片 4）
- worker_C 优先级 10 → Q2（时间片 8）

### 3.2 自动调度 —— 观察完整降级链

```bash
start_sched
```

每 2秒 自动输出一行调度日志。约 10~15 秒跑完，关键输出：

```text
调度器已启动 (多级反馈队列)
Q0(优先级 0-3): worker_A(2)
Q1(优先级 4-7): worker_B(3)
Q2(优先级 8-15): worker_C(4)

[调度] >> 选中 PID=2 (worker_A) 来自 Q0(优先级 0-3) | 时间片=2 | CPU=2/10
```

> Q0→Q1 降级，worker_A 用完时间片排到 Q1 末尾

```text
[调度] >> 选中 PID=2 (worker_A) 来自 Q1(优先级 4-7) | 时间片=4 | CPU=6/10
```

> Q0 空 → Q1 取 worker_A → 时间片=4 → 6<10 → 降级到 Q2

```text
[调度] >> 选中 PID=3 (worker_B) 来自 Q1(优先级 4-7) | 时间片=4 | CPU=4/10
```

> Q0 空 → Q1 取 worker_B → 时间片=4 → 4<10 → 降级到 Q2

```text
[调度] >> 选中 PID=2 (worker_A) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=14/10
```

> Q0、Q1 均空 → Q2 队首 worker_A → 时间片=8 → 14<10 → 已在 Q2 不降

```text
[调度] >> 选中 PID=3 (worker_B) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=12/10
[调度] >> 选中 PID=4 (worker_C) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=8/10
[调度] >> 选中 PID=2 (worker_A) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=22/10
CPU 22 >= 10 → 进程完成! 已自动终止
[调度] >> 选中 PID=3 (worker_B) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=20/10
CPU 20 >= 10 → 进程完成! 已自动终止
[调度] >> 选中 PID=4 (worker_C) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=16/10
[调度] >> 选中 PID=4 (worker_C) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=24/10
CPU 24 >= 10 → 进程完成! 已自动终止

[调度] 所有用户进程已完成, 调度器自动停止
```

**验收要点**：

- worker_A 经历了 **Q0 → Q1 → Q2** 完整三次降级
- worker_B 从 Q1 → Q2，worker_C 一直在 Q2
- 三个进程交替运行，时间片大的等得久但一次跑得多
- CPU 数值在动态增长，不是直接改状态

### 3.3 单步演示 —— 完整决策链路

> 核心要求：每一步必须展示「队列当前状态 → 选中谁 → 决策逻辑 → 执行结果」的完整推导。

清空旧进程，创建新测试进程。burstTime=6 确保多步才能跑完：

```bash
kill_pcb 2
kill_pcb 3
kill_pcb 4
create_pcb s_a 3 1 6
create_pcb s_b 8 1 6
```

s_a 优先级 3→Q0（时间片 2），s_b 优先级 8→Q2（时间片 8）。

#### 第一步

```bash
step
```

```text
当前队列状态:
  Q0(优先级 0-3): s_a(2)[0/6]
  Q1(优先级 4-7): (空)
  Q2(优先级 8-15): s_b(3)[0/6]
[调度] >> 选中 PID=2 (s_a) 来自 Q0(优先级 0-3) | 时间片=2 | CPU=2/6
   决策: Q0 非空 → 取 s_a → 时间片 2 | CPU 2 < 6 → 时间片耗尽, 降级 Q0→Q1
   执行后队列: Q1=s_a(2)[2/6] | Q2=s_b(3)[0/6]
```

> **决策推导**：扫描 Q0=[s_a], Q1=[], Q2=[s_b]。Q0 非空 → 取 s_a → 时间片 2 → CPU 2 < 6 未完成 → 降级到 Q1。现在 Q1=[s_a], Q2=[s_b]。

#### 第二步

```bash
step
```

```text
当前队列状态:
  Q0(优先级 0-3): (空)
  Q1(优先级 4-7): s_a(2)[2/6]
  Q2(优先级 8-15): s_b(3)[0/6]
[调度] >> 选中 PID=2 (s_a) 来自 Q1(优先级 4-7) | 时间片=4 | CPU=6/6
   决策: Q1 非空 → 取 s_a → 时间片 4 | CPU 6 >= 6 → 进程完成! 已自动终止
   执行后队列: Q2=s_b(3)[4/6]
```

> **决策推导**：Q0=[], Q1=[s_a], Q2=[s_b]。Q1 队首 s_a → 时间片 4 → CPU 6 ≥ 6 → 完成！s_a 自动终止。队列剩下 Q2=[s_b]。

#### 第三步

```bash
step
```

```text
当前队列状态:
  Q0(优先级 0-3): (空)
  Q1(优先级 4-7): (空)
  Q2(优先级 8-15): s_b(3)[4/6]
[调度] >> 选中 PID=3 (s_b) 来自 Q2(优先级 8-15) | 时间片=8 | CPU=8/6
   决策: Q2 非空 → 取 s_b → 时间片 8 | CPU 8 >= 6 → 进程完成! 已自动终止
   执行后队列: (全空)
```

> s_b 在一轮时间片 8 内直接跑完（只需 6），三步骤调度演示完毕。

#### 第四步

```bash
step
```

```text
所有调度队列为空, 无可调度进程
```

#### 第五步

```bash
step
```

#### 第六步


## 八、异常/边界测试

### 8.1 非法参数

```bash
create_pcb              → 用法: create_pcb <进程名> <优先级(0-15)> [父进程PID] [需要CPU时间]
create_pcb x 20         → (优先级20会被截断为15, 正常创建)
renice 999 5            → 修改失败: PID不存在或优先级范围无效(0-15)
kill_pcb 999            → 撤销失败: PID=999 不存在或为init进程
alloc -1 1              → 大小必须在1-1024KB之间
```

### 8.2 边界值

```bash
create_pcb edge_a 0         → 优先级0（最高）
create_pcb edge_b 15        → 优先级15（最低）
```

预期输出：

```text
进程创建成功! PID=17 名称=edge_a 优先级=0 需要CPU=5
进程创建成功! PID=18 名称=edge_b 优先级=15 需要CPU=5
```

清理：

```bash
kill_pcb 17
kill_pcb 18
```

### 8.3 kill init

```bash
kill_pcb 1
```

预期输出：

```text
撤销失败: PID=1 不存在或为init进程
```
