# 操作系统模拟器 测试流程


## 一、用户管理

### 1.1 注册新用户

```
register admin 123456
```

预期输出：
```
[OK] 用户 admin 注册成功
```

### 1.2 重复注册

```
register admin 123456
```

预期输出：
```
[ERROR] 用户 admin 已存在
```

### 1.3 登录

```
login admin 123456
```

预期输出：
```
[OK] 欢迎 admin，登录成功
```

### 1.4 错误密码

```
login admin 111111
```

预期输出：
```
[ERROR] 密码错误
```

### 1.5 未登录执行命令

```
logout
create_pcb test 5
```

预期输出：
```
[OK] 已登出
[ERROR] 请先登录
```

## 二、进程管理

重新登录后开始：

```
login admin 123456
```

### 2.1 创建进程

```
create_pcb proc_a 3
create_pcb proc_b 7
create_pcb proc_c 1
```

预期输出（PID 从 2 开始，1 是系统 init 进程）：
```
进程创建成功! PID=2 名称=proc_a 优先级=3 需要CPU=5
进程创建成功! PID=3 名称=proc_b 优先级=7 需要CPU=5
进程创建成功! PID=4 名称=proc_c 优先级=1 需要CPU=5
```

### 2.2 创建子进程

```
create_pcb child_a 5 2 3
create_pcb child_b 6 2 2
```

预期输出：
```
进程创建成功! PID=5 名称=child_a 优先级=5 需要CPU=3
进程创建成功! PID=6 名称=child_b 优先级=6 需要CPU=2
```

### 2.3 列出所有进程

## 四、内存管理

### 4.1 创建测试进程

```
create_pcb mem_test 5 1
```

### 4.2 查看初始内存

```
show_mem
```

预期输出：
```
内存总量: 1024KB
空闲: 1024KB
空闲块列表:
  起始=0KB, 大小=1024KB
```

```
alloc 100 9

```

```
[OK] PID=9 分配 100KB 于地址 0KB
[OK] PID=9 分配 200KB 于地址 100KB
```

```
free_mem 100

```

```

已分配:
  0KB-100KB: PID=9
```


## 五、持久化

### 5.1 保存状态

创建一些进程和分配内存：

```
create_pcb save_a 3
create_pcb save_b 5
alloc 128 10
alloc 256 10
save
```

预期输出：
```
[OK] 系统状态已保存至 数据/os_state.bin
```

### 5.2 验证

重启程序后：

```
login admin 123456
list_pcb
```

预期输出（恢复后的状态与保存前一致）：
```
PID  名称      优先级  PPID  状态     CPU已用/需
--------------------------------------------------
10   save_a    3      0      就绪     0/5
11   save_b    5      0      就绪     0/3
```

```
show_mem
```

预期输出（内存分配恢复）：
```
已分配:
  0KB-128KB: PID=10
  128KB-384KB: PID=10
```

### 5.3 清理

```
kill_pcb 10
kill_pcb 11
save
```

## 七、IPC 多实例（可选）

启动两个终端：

```
# 终端1（后端）
操作系统模拟器.exe
> login admin 123456
> create_pcb ipc_test 5
> list_pcb

# 终端2（观察者）
操作系统模拟器.exe
> login admin 123456
> list_pcb
```

终端2 预期输出（与终端1 一致）：
```
PID  名称      优先级  PPID  状态     CPU已用/需
--------------------------------------------------
14   ipc_test  5      0      就绪     0/3
```

终端2 中执行：
```
create_pcb ipc_test2 3 0 2
list_pcb
```

终端1 中验证：
```
list_pcb
```

预期输出（终端1 也能看到终端2 创建的进程）：
```
PID  名称        优先级  PPID  状态     CPU已用/需
--------------------------------------------------
14   ipc_test    5      0      就绪     0/3
15   ipc_test2   3      0      就绪     0/2
```

## 八、异常测试

### 8.1 非法参数

```
create_pcb
create_pcb x 20
renice 999 5
kill_pcb 999
alloc -1 1
```

预期输出（均为错误提示，程序不崩溃）：
```
[ERROR] 参数不足: create_pcb <名称> <优先级> [ppid] [需CPU]
[ERROR] 优先级范围 0-15
[ERROR] PID=999 不存在
[ERROR] PID=999 不存在
[ERROR] 分配大小必须为正数
```

### 8.2 边界值

```
create_pcb edge_a 0
create_pcb edge_b 15
```

预期输出：
```
[OK] 进程 edge_a 已创建, PID=16, PPID=1, 优先级=0
[OK] 进程 edge_b 已创建, PID=17, PPID=1, 优先级=15
```

```
kill_pcb 16
kill_pcb 17
```
