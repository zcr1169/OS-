## 1. kill_pcb — 级联撤销怎么实现

### 核心思想

用 **递归** 的方式先杀子进程，再杀自己。这样杀掉父进程时，它的所有子孙进程都会被一并清理，不会产生"孤儿进程"。

### 数据结构准备

每个 PCB 中维护了一个 `children` 列表：

```cpp
struct PCB {
    int32_t pid;
    int32_t ppid;           // 父进程 PID，-1 表示无父
    std::vector<int32_t> children;  // 子进程 PID 列表
};
```

创建进程时把自己加到父进程的 children 中：

```cpp
int32_t pid = nextPid_++;
PCB pcb(pid, ppid, name, PCB::READY, ...);
pcbs_[pid] = pcb;
if (ppid >= 0) {
    pcbs_[ppid].children.push_back(pid);  // 加入父进程子列表
}
```

### killPCB 函数

```cpp
bool ProcessManager::killPCB(int32_t pid,
                             std::function<void(int32_t)> onKill) {
    // 1) 从父进程的 children 中移除自己
    // 2) 递归杀子进程 → 回调释放调度队列和内存
    // 3) 从哈希表删除自己
    killChildren(pid, onKill);  // 先杀子
    if (onKill) onKill(pid);    // 回调释放自己的资源
    pcbs_.erase(pid);           // 从哈希表删除
    return true;
}
```

### killChildren 递归

```cpp
void ProcessManager::killChildren(int32_t pid, ...) {
    auto children = it->second.children;  // 复制一份
    for (int32_t childPid : children) {
        killChildren(childPid, onKill);    // 递归杀孙子
        if (onKill) onKill(childPid);      // 释放孙子资源
        pcbs_.erase(childPid);             // 删除孙子
    }
}
```

### 调用方

```cpp
processMgr_.killPCB(pid, [&](int32_t killedPid) {
    scheduler_.dequeue(killedPid);    // 从调度队列移除
    memoryMgr_.freeByPid(killedPid);  // 释放内存
});
```

## 2. ptree — 树形结构显示进程父子关系怎么实现

### 核心思想

利用 PCB 中的 `ppid` 和 `children` 字段，用 **DFS 递归** 遍历进程树，每深入一层增加缩进。

```cpp
std::string ProcessManager::pTree(const std::string& owner) const {
    std::string out;
    // 第一遍：找根进程（ppid==-1），从根开始递归
    for (const auto& pair : pcbs_) {
        const PCB& p = pair.second;
        if (p.ppid == -1 && (owner.empty() || p.owner == owner))
            pTreeRecursive(p.pid, 0, owner, out);
    }
    // 第二遍：处理父进程被过滤掉的孤儿进程
    // ...
    return out;
}
```

### 递归画树

```cpp
void ProcessManager::pTreeRecursive(int32_t pid, int depth,
                                     const std::string& owner,
                                     std::string& out) const {
    // 画缩进：深度越深缩进越多
    for (int i = 0; i < depth; i++)
        out += (i == depth - 1) ? "├─ " : "   ";

    // 输出进程信息
    out += p.name + "(" + std::to_string(p.pid) + ")";
    out += " [" + std::string(PCB::stateToString(p.state))
           + ", 优先级=" + std::to_string(p.priority)
           + ", 内存=" + std::to_string(p.totalMemory) + "KB]\n";

    // 递归遍历子进程
    for (size_t i = 0; i < p.children.size(); i++)
        pTreeRecursive(p.children[i], depth + 1, owner, out);
}
```

### 输出效果

```
proc_a(2) [就绪, 优先级=3, 内存=0KB]
├─ child_a(5) [就绪, 优先级=5, 内存=0KB]
├─ child_b(6) [就绪, 优先级=6, 内存=0KB]
```

## 3. 多线程和多实例是怎么实现的

### 三线程架构

```
┌─ 前台线程（main 循环）
│    getline(cin) 读输入 → parse → cmdQueue.push
│    ← resultQueue.pop(5s) 等结果 → cout 输出
│    每命令后 save() 持久化
│
├─ 后台线程（backendLoop）
│    cmdQueue.pop 取命令 → executeCommand
│    → resultQueue.push 返回结果
│    检查观察者的 commands.txt
│
└─ 调度器线程（schedulerLoop）
     每 2 秒调 step() 执行一次调度
     连续 3 次空队列自动停止
```

### 消息队列（前后台解耦）

```cpp
template <typename T>
class MessageQueue {
    std::queue<T> queue_;
    std::mutex mtx_;
    std::condition_variable cv_;

    void push(const T& msg) {
        { lock_guard<mutex> lock(mtx_); queue_.push(msg); }
        cv_.notify_one();    // 通知消费者
    }

    bool pop(T& msg, int timeoutMs) {
        unique_lock<mutex> lock(mtx_);
        cv_.wait_for(lock, timeoutMs,
            [this] { return !queue_.empty() || stopped_; });
        if (queue_.empty()) return false;
        msg = std::move(queue_.front()); queue_.pop();
        return true;
    }
};
```

### 线程创建

```cpp
// 后台线程
isBackend_ = backendLock_.tryLock(lockFilePath());
if (isBackend_) {
    backendThread_ = std::thread(&OSSimulator::backendLoop, this);
}
// 调度器线程（在 start_sched 时启动）
schedThread_ = std::thread(&Scheduler::schedulerLoop, this);
```

### 多实例：后端 vs 观察者

```
实例A（先启动）→ tryLock 拿到文件锁 → 成为"后端"
  ├─ 拥有后台线程 + 调度器线程
  └─ 直接执行命令，每命令后 save()

实例B（后启动）→ tryLock 失败 → 成为"观察者"
  ├─ 没有后台线程
  ├─ 命令写入 commands.txt
  └─ pollLoop 每 200ms 轮询 os_state.bin
```

### 文件锁实现

```cpp
bool FileLock::tryLock(const std::string& filePath) {
    // Windows: CreateFileW + LockFileEx
    handle_ = CreateFileW(path, ...);
    if (!LockFileEx(handle_,
            LOCKFILE_EXCLUSIVE_LOCK | LOCKFILE_FAIL_IMMEDIATELY,
            0, 1, 0, &ov)) {
        return false;  // 锁被占着
    }
    return true;  // 拿到锁
}
```

### 观察者轮询

```cpp
void OSSimulator::pollLoop() {
    while (running_.load()) {
        // 每次循环先试能不能升级为后端
        if (backendLock_.tryLock(lockFilePath())) {
            isBackend_ = true;
            load();  // 加载最新状态
            backendThread_ = std::thread(backendLoop, this);
            return;  // poll 线程退出，主循环走后端路径
        }
        // 检查 os_state.bin 时间戳
        auto ftime = last_write_time(stateFilePath());
        if (ftime != lastPollTime_) {
            lastPollTime_ = ftime;
            load();  // 重新加载状态
        }
        sleep(200ms);
    }
}
```

### 观察者发命令

```cpp
// 观察者：写入 commands.txt
ofstream f("commands.txt", ios::app);
f << input << "\n";

// 后端：定时检查并执行
void checkObserverCommands() {
    ifstream f("commands.txt");
    for (auto& line : lines) {
        auto cmd = CommandParser::parse(line);
        if (!cmd.name.empty()) executeCommand(cmd);
    }
    save();  // 保存状态供观察者同步
}
```

## 4. 多级反馈队列怎么入队、降级、调度、结束

### 数据结构

```cpp
std::vector<std::deque<int32_t>> queues_[3];
// 三个双端队列，存 PID
const int TIME_SLICE[3] = {2, 4, 8};  // Q0=2, Q1=4, Q2=8
```

### 入队规则

```cpp
void Scheduler::enqueue(int32_t pid, int32_t priority) {
    int qIdx = priorityToQueue(priority);  // 0→Q0, 5→Q1, 10→Q2
    // 先移除旧记录（防重复）
    queues_[qIdx].push_back(pid);  // 加到队尾
}
```

不同场景的入队参数：

| 场景 | 代码 | 去向 |
|------|------|------|
| 创建进程 | `enqueue(pid, 0)` | **Q0** |
| 唤醒进程 | `enqueue(pid, 0)` | **Q0** |
| 恢复挂起 | `enqueue(pid, 0)` | **Q0** |
| 启动调度 | `enqueue(pid, 0)` | **Q0** |
| renice | `enqueue(pid, newPrio)` | 按新优先级 |

### 降级规则

```cpp
int nextQ = min(qIdx + 1, 2);               // 先降一级
int prioQ = priorityToQueue(pcb->priority);  // 再按优先级
int targetQ = max(nextQ, prioQ);             // 取较高级
queues_[targetQ].push_back(pid);
```

| 当前队列 | 优先级 | 结果 |
|---------|--------|------|
| Q0 | 2 | 降 Q1 |
| Q0 | 10 | 降 Q2 |
| Q1 | 5 | 降 Q2 |
| Q2 | 10 | 不降，原地继续 |

### 调度流程（step 函数）

```
① 打印当前队列快照
② 按 Q0→Q1→Q2 顺序扫描，取第一个非空队列的队首
③ 队首出队 → RUNNING → cpuTime += 时间片
   ┌─ cpuTime ≥ burstTime → TERMINATED + 释放内存
   └─ 否则 → READY + 降级到下一队列
④ 输出执行后队列状态
```

### 进程结束

**自动终止**（step 中判断）：

```cpp
if (pcb->cpuTime >= pcb->burstTime) {
    pcb->state = PCB::TERMINATED;
    if (onTerminate_) onTerminate_(pid);  // 释放内存
    pcb->memoryBlocks.clear();
    pcb->totalMemory = 0;
    // 该进程不再入队，自然消失
}
```

**kill_pcb 手动杀死**：

```cpp
processMgr_.killPCB(pid, [&](int32_t killedPid) {
    scheduler_.dequeue(killedPid);     // 从队列移除
    memoryMgr_.freeByPid(killedPid);   // 释放内存
});
```

### 调度线程自动停止

```cpp
void Scheduler::schedulerLoop() {
    int emptyRounds = 0;
    while (!stopRequested_) {
        string log = step();
        if (log 含 "所有调度队列为空") {
            emptyRounds++;
            if (emptyRounds >= 3) {
                running_ = false; break;  // 自动停止
            }
        } else {
            cout << log;
            emptyRounds = 0;
        }
        cv_.wait_for(lock, 2000ms);  // 每 2 秒调度一次
    }
}
```

## 总结

| 模块 | 核心数据结构 | 算法/机制 |
|------|-------------|----------|
| 进程管理 | `unordered_map<PID, PCB>` | 递归级联删除 |
| 进程树 | PCB 的 `children` 列表 | DFS 递归 + 缩进 |
| 多线程 | `MessageQueue` + `mutex` + `cv` | 生产者消费者 |
| 多实例 | `FileLock` + 文件轮询 | 后端/观察者模式 |
| MLFQ 调度 | `vector<deque<PID>>` | 时间片 2→4→8，用完降级 |
| 自动停止 | `emptyRounds ≥ 3` | 空队列自动检测 |
