## 快速开始

直接运行 `生成/OSCD.exe`，首次使用先注册再登录：

```text
register admin 123456
login admin 123456
```text

### 多实例并发

先启动第一个终端（后端），再启动第二个终端（观察者）：

```text
# 终端1 - 后端（先启动，自动获取文件锁）
OSCD.exe

# 终端2 - 观察者（后启动，命令通过文件传递到后端）
OSCD.exe
```text

观察者所有命令通过 `commands.txt` 文件传递到后端执行，实时共享同一份内核状态。
双窗口验证：A端 `create_pcb test 5` → B端 `list_pcb` 立即可见新进程。

## 命令列表

### 用户管理
| 命令 | 说明 |
|------|------|
| `register <用户名> <密码>` | 注册新用户 |
| `login <用户名> <密码>` | 登录 |
| `logout` | 登出 |

### 进程管理
| 命令 | 说明 |
|------|------|
| `create_pcb <名称> <优先级> [ppid] [需CPU]` | 创建进程 (优先级0-15, 0最高, 默认需5个CPU时间) |
| `kill_pcb <PID>` | 撤销进程 (级联撤销子进程) |
| `block_pcb <PID>` | 阻塞进程 |
| `wakeup_pcb <PID>` | 唤醒进程 |
| `show_pcb <PID>` | 查看单个PCB详细信息 |
| `list_pcb` | 列出所有进程及状态 |
| `ptree` | 以树形结构显示进程父子关系 |
| `suspend <PID>` | 挂起进程（从调度队列移出） |
| `resume <PID>` | 恢复挂起进程 |
| `renice <PID> <优先级>` | 修改进程优先级 |

### 调度器
| 命令 | 说明 |
|------|------|
| `start_sched` | 启动后台调度线程，开始按多级反馈队列自动调度 |
| `stop_sched` | 暂停调度 |
| `restart_sched` | 重启调度 |
| `step` | 单步执行一次调度，选中进程运行一个时间片（模拟执行时长消耗） |

### 内存管理
| 命令 | 说明 |
|------|------|
| `alloc <大小KB> <PID>` | 手工申请一块指定大小的内存（用于模拟内核分配） |
| `free_mem <起始地址KB>` | 手工释放一块已分配的内存 |
| `show_mem` | 显示当前内存分区使用情况（空闲块链表+已分配块列表） |
| `compact` | 执行内存紧缩，合并相邻空闲分区 |
| `mem_stat` | 统计内存碎片率、空闲总量等 |
| `set_alloc_algo <FF\|BF\|WF>` | 切换分配算法（首次适应/最佳适应/最坏适应） |
| `pgfault` | 模拟一次缺页中断（可仅打印提示） |
| `swap_out <PID>` | 将某进程的部分内存标记为换出（不计入实际磁盘读写） |

### 持久化 & 可视化
| 命令 | 说明 |
|------|------|
| `save` | 将当前所有进程、调度队列、内存分配状态保存至二进制物理文件 |
| `load` | 从该物理文件恢复全部状态，程序启动时若检测到该文件则自动加载，否则从零初始状态开始 |
| `overview` | 系统全景快照 (进程树+内存图+调度队列+统计) |

## 技术要点

### 多级反馈队列调度器
- **Q0**: 优先级 0-3, 时间片 1
- **Q1**: 优先级 4-7, 时间片 2
- **Q2**: 优先级 8-15, 时间片 4
- 用完时间片自动降级，定期老化防饥饿

### 内存分配算法
- **首次适应 (FF)**: 选第一个够大的空闲块
- **最佳适应 (BF)**: 选最接近请求大小的空闲块
- **最坏适应 (WF)**: 选最大的空闲块

### 多线程架构
```text
前台线程 ──cmdQueue──▶ 后台线程 ──▶ 命令处理
                           │
                    schedulerLoop (独立线程)

观察者A ──commands.txt──┐
                      ├──▶ 后端 checkObserverCommands() → executeCommand
观察者B ──commands.txt──┘
```text
